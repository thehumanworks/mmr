## Messages Ordering and Pagination

Normal `messages` queries filter first, sort second, apply any message-index range third, and
paginate last.

For the default `timestamp asc` path, pagination still selects a newest-first window and then
reverses it back to chronological order:

```rust
let descending = filtered.into_iter().rev().collect::<Vec<_>>();
let mut paged = apply_pagination(descending, options.limit, options.offset);
paged.reverse();
```

`next_command` is generated by the CLI layer only when `next_page` is `true`, and it preserves
the active `--source`, `--pretty`, `--session`, `--project`, `--all`, message-index range,
`--limit`, `--sort-by`, and `--order` flags.

Source: `src/messages/service.rs:319-332`, `src/cli.rs:257-269`, `src/cli.rs:480-541`

## Latest-Session and Message-Index Semantics

`mmr messages --latest [N]`:

- selects the latest session within the current filter scope
- sorts that session chronologically
- applies any message-index range
- returns the newest `N` messages from that session in chronological order

Latest-session responses always set `next_page = false` and `next_command = None`.

Message-index range rules:

- `--from-message-index` is inclusive
- `--to-message-index` is exclusive
- the range is applied after filtering and sorting
- out-of-range bounds clamp to the available length
- the CLI rejects `from > to`

Source: `src/messages/service.rs:360-422`, `src/messages/service.rs:688-706`, `src/cli.rs:383-392`

## Session Lookup Semantics

When `mmr messages --session <ID>` is called without `--project`, the CLI skips cwd project
auto-discovery and searches all projects. If `--source` is also omitted, it prints this hint to
`stderr`:

```text
hint: searching all sources for session; pass --source to narrow the search
```

## Codex Project Normalization

Project resolution accepts Codex-style paths either with or without a leading slash:

```rust
if trimmed.starts_with('/') {
    let without_leading = trimmed.trim_start_matches('/');
    if !without_leading.is_empty() {
        candidates.push(without_leading.to_string());
    }
} else {
    candidates.push(format!("/{trimmed}"));
}
```

The resolver searches Codex by canonical/original path matches and lets cross-source queries
resolve Claude/Cursor project encodings through the same shared helper.
